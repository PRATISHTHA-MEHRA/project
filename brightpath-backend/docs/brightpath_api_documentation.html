BrightPath Admin — every module, endpoint, and auth requirement in one reference.

🔑 Authentication Requirements

Auth Token
Required for most endpoints in this doc. Send as an Authorization: Bearer <token> header. The token is a JWT, stored client-side in localStorage as JWT_TOKEN, and identifies the signed-in admin user. Routes marked Auth Required below use this.
No Auth
Routes with no auth middleware attached. Currently: POST /api/auth/login (to obtain a token in the first place), GET /api/timetable, and every route in the Homework, Settings, and Reports modules — none of those three modules' routes have the auth middleware wired in, unlike every other module. Worth double-checking that's intentional; marked No Auth below either way, since that's what the route files actually show.
API Key
Not currently used by any route in this backend — every protected route relies on the JWT auth middleware only. Kept here in case a front-facing/guest surface is added later that needs a separate api-key header or query parameter.
